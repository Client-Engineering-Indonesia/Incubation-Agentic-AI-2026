# Module 2.2: Install ADK

---

## Overview

In this module, you'll learn how to install and configure the **watsonx Orchestrate Agent Development Kit (ADK)** on your local machine. The ADK is a Python-based SDK that enables you to build, test, and deploy AI agents programmatically.

**Learning Objectives:**
- Install Python and required dependencies
- Install the watsonx Orchestrate ADK package
- Set up a virtual environment
- Configure ADK with your credentials
- Verify the installation
- Set up your development environment

## System Requirements

### Supported Operating Systems

- **macOS**: 10.15 (Catalina) or later
- **Linux**: Ubuntu 20.04+, RHEL 8+, or equivalent
- **Windows**: Windows 10/11 with WSL2 (recommended) or native Windows

### Software Requirements

- **Python**: 3.9, 3.10, or 3.11 (3.11 recommended)
- **pip**: Latest version
- **Git**: 2.x or later (optional but recommended)
- **Code Editor**: VS Code, PyCharm, or similar

### Hardware Requirements

- **RAM**: Minimum 4GB, 8GB+ recommended
- **Disk Space**: At least 2GB free space
- **Processor**: Modern multi-core processor

## Part 1: Install Python

### Step 1: Check Python Installation

First, check if Python is already installed:

```bash
# Check Python version
python --version
# or
python3 --version
```

**Expected Output:**
```
Python 3.11.x
```

If Python is not installed or version is below 3.9, proceed with installation.

---

### Step 2: Install Python (if needed)

#### On macOS

**Option 1: Using Homebrew (Recommended)**

```bash
# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Python
brew install python@3.11

# Verify installation
python3.11 --version
```

**Option 2: Download from python.org**

1. Visit https://www.python.org/downloads/
2. Download Python 3.11.x for macOS
3. Run the installer
4. Follow installation wizard

---

#### On Linux (Ubuntu/Debian)

```bash
# Update package list
sudo apt update

# Install Python 3.11
sudo apt install python3.11 python3.11-venv python3.11-dev

# Install pip
sudo apt install python3-pip

# Verify installation
python3.11 --version
pip3 --version
```

---

#### On Windows

**Option 1: Using Windows Subsystem for Linux (WSL2) - Recommended**

```bash
# Install WSL2 (in PowerShell as Administrator)
wsl --install

# After restart, open Ubuntu and install Python
sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip
```

**Option 2: Native Windows Installation**

1. Visit https://www.python.org/downloads/windows/
2. Download Python 3.11.x for Windows
3. Run installer
4. ‚úÖ **Important**: Check "Add Python to PATH"
5. Click "Install Now"

---

## Part 2: Set Up Virtual Environment

### Step 4: Create Project Directory

Create a dedicated directory for your ADK projects:

```bash
# Create project directory
mkdir -p ~/watsonx-adk-projects
cd ~/watsonx-adk-projects

# Create a specific project folder
mkdir my-first-agent
cd my-first-agent
```

---

### Step 5: Create Virtual Environment

Create an isolated Python environment for your project:

```bash
# Create virtual environment
python3 -m venv venv

# Alternative: specify Python version
python3.11 -m venv venv
```

**What is a Virtual Environment?**
- Isolated Python environment
- Separate package installations
- Prevents dependency conflicts
- Easy to recreate and share

---

### Step 6: Activate Virtual Environment

Activate the virtual environment:

**On macOS/Linux:**
```bash
source venv/bin/activate
```

**On Windows (Command Prompt):**
```cmd
venv\Scripts\activate
```

**On Windows (PowerShell):**
```powershell
venv\Scripts\Activate.ps1
```

**Verification:**
Your prompt should now show `(venv)` prefix:
```
(venv) user@machine:~/watsonx-adk-projects/my-first-agent$
```

## Part 3: Install watsonx Orchestrate ADK

### Step 7: Install ADK Package

With your virtual environment activated, install the ADK:

```bash
# Install watsonx Orchestrate ADK
pip install ibm-watsonx-orchestrate-adk

# Or install with specific version
pip install ibm-watsonx-orchestrate-adk==1.0.0
```

## Part 4: Configure ADK

### Step 10: Set Up Configuration File

Create a configuration file for your ADK credentials:

```bash
# Create .env file for credentials
touch .env

# Open in your editor
nano .env
# or
code .env
```

**Add the following to .env:**
```bash
# watsonx Orchestrate Credentials
WATSONX_ORCHESTRATE_API_KEY=your_api_key_here
WATSONX_ORCHESTRATE_INSTANCE_ID=your_instance_id_here
WATSONX_ORCHESTRATE_REGION=us-south
WATSONX_ORCHESTRATE_URL=https://api.watson-orchestrate.ibm.com

# Optional: Project/Space IDs
WATSONX_PROJECT_ID=your_project_id
WATSONX_SPACE_ID=your_space_id

# Development Settings
ENVIRONMENT=development
LOG_LEVEL=INFO
```
